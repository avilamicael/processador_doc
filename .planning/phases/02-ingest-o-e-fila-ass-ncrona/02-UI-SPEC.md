# Phase 2 — UI Design Contract

> Visual and interaction contract for **Ingestão e Fila Assíncrona**. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **LOCKED DESIGN SYSTEM.** A complete, user-approved visual design ("DocWatch — Opção B / Corporate Modern") already exists and is implemented in `frontend/`. This spec does **not** invent a new visual language — it adapts the existing tokens, primitives, and component classes (`frontend/src/index.css`, `frontend/src/components/`) to the **real Phase-2 data, states, and scope**. All tokens below are transcribed from `frontend/src/index.css`; do not re-derive or override them.

## Spacing Scale

The locked CSS uses an effectively 4-based rhythm with a few design-specific odd values inherited from the original bundle (e.g. `--row-py: 13px`, card padding `15–18px`). **Reuse the existing component classes and their padding as-is** — do not re-pad with new arbitrary values. For any NEW element introduced this phase (e.g. empty-state block, duplicate banner, folder form), use the scale below.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section padding (`.scroll` uses 24px) |
| xl | 32px | Layout gaps, empty-state vertical padding |
| 2xl | 48px | Empty-state block padding (table body) |
| 3xl | 64px | Page-level spacing |

Exceptions (inherited from locked design — keep, do not "fix"): `--row-py: 13px` (table cell vertical padding), card paddings `15px/16px/18px`, toolbar `13px 16px`, list-head `14px 18px`, folder-row `14px 18px`. New code must not introduce additional off-scale values beyond these.

## Color

Transcribed from `:root` (light) in `frontend/src/index.css`; each has a `[data-theme="dark"]` counterpart. Use `var(--…)` always.

| Role | Value (light) | Usage |
|------|---------------|-------|
| Dominant (60%) | `--bg` `#F5F7FB` | App background, page surface |
| Secondary (30%) | `--surface` `#FFFFFF`, `--surface-2` `#F8FAFD`, `--surface-3` `#EFF2F8` | Cards, table header, sidebar, nav, inputs |
| Accent (10%) | `--accent` `#2563EB` (dark: `#3B82F6`) | See reserved list below |
| Destructive | `--st-erro` `#DC2626` (dark: `#F87171`) | FALHA status pill, delete-folder confirmation only |

**Accent (`--accent`) reserved EXCLUSIVELY for:**
- Primary buttons (`.btn-primary`) — "Adicionar pasta", "Forçar varredura"
- Active nav item + active-state indicator (`.nav-item.active`, `.nav-ind`)
- Active filter chip background/text (`.chip.active`, via `--accent-soft`)
- Active tab underline (`.tab.active`)
- Selected table row tint (`tr.selected`, via `--accent-soft`)
- Focus ring on inputs (`--accent-ring`)
- Folder icon tile (`.folder-icon`, via `--accent-soft`)
- Switch "on" state (`.switch.on`)
- Active pagination page (`.pg.active`)

Accent is **never** used for body text, borders of static cards, or as a generic "interactive" color on every element.

## Interaction & State Contract

### Documentos (list)

- **Data source:** real backend documents (the **blocks/Documents** of D-06), not duplicates. Duplicates never appear as rows (D-10) — only the aggregate indicator.
- **Refresh:** **polling via TanStack Query** (`refetchInterval`, recommend 3–5s while the page is focused; pause when tab hidden via `refetchIntervalInBackground: false`). New documents appear/transition without manual reload — satisfies success criterion "vê os documentos entrando na fila".
- **Polling must not flicker:** keep previous data during refetch (`keepPreviousData`/`placeholderData`); never collapse the table to a spinner on background refetch.
- **Columns in scope (v1):** Arquivo (mono name + `docMini` icon), Pasta de origem (mono path), Status (pill, real domain state), Tamanho (mono), Data. **Remove/hide:** Tipo, Template, Responsável (later phases / removed from v1 — README). Row actions: keep visual but only wire what exists this phase (no extraction/automation actions); a non-functional action must be hidden, not shown disabled with no purpose.
- **Status chips/filters:** relabel chip set to real domain states (Na fila / Processando / Aguardando extração / Falha). Counts come from the backend. "Forçar varredura" triggers a manual rescan endpoint; show it as `.btn-primary` with `refresh` icon (existing).
- **Duplicate indicator placement:** a single, unobtrusive element — recommended in the `.table-foot` or as a `.badge`-style chip near the toolbar — using `--text-3` neutral treatment (NOT accent, NOT error; ignored duplicates are normal/routine per D-03/D-10). Must not be styled as an alert.
- **Empty state:** when zero documents, render the empty-state copy centered inside the table card body (replace `<tbody>` rows), not a blank table.

### Configurações → Pastas monitoradas (CRUD)

- **List:** reuse `.folder-row` + `.folder-icon`. Per-row meta (`.folder-meta`) shows: split rule ("Separar a cada N páginas" or "Não separar") + file/ingested count if available. **Drop** the mock's "Recursiva / tipos / freq" meta unless the backend exposes them; do not display placeholder/mock values.
- **Add/Edit form:** modal or inline panel reusing existing tokens (`.card`, `.search-input`/`.select` styling for fields, `.btn-primary`/`.btn-ghost` for actions). Fields per the table above. Persist to DB (D-02).
- **Remove:** destructive confirmation copy above; the confirm button uses `--st-erro` color. Removing a folder stops monitoring only — never deletes ingested documents (D-03).
- **Per-folder toggle:** reuse `.switch` for activate/deactivate.
- **Watcher status:** the existing sidebar `.watcher-box` and the `PastasTab` `.list-head` "Watcher global" indicator may reflect real watcher running-state; keep visual, wire to real status if available, otherwise leave static (do not fabricate "última varredura: há 2 min" mock text).

### Accessibility / touch

- Interactive controls keep visible focus via `--accent-ring` (already on inputs). Icon-only buttons (`.icon-btn`, `.row-action`) must have `aria-label`/`title` (pattern already present in `DocumentsPage.tsx`).
