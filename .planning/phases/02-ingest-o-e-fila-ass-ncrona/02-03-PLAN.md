## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Job payload (JSON) → worker | payload carrega source_path; deve referir-se a pasta monitorada conhecida |
| Worker → DB (single-writer) | claim/escritas concorrentes; corrida de dois consumidores |
| Crash/restart → fila | jobs em running precisam resumir sem duplicar trabalho/cobrança |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering / Elevation | source_path no payload aponta para fora das pastas monitoradas | mitigate | payload é gerado internamente (não vem do browser); o ingest_stage só processa caminhos vindos do watcher/rescan (Plano 04 valida sob pasta conhecida); allowlist de extensão reduz superfície (ING-04) |
| T-02-07 | Denial of Service / Repudiation | retry storm em falha transitória | mitigate | backoff exponencial+jitter + dead-letter→FALHA após max_attempts (Pattern 2); teste de backoff |
| T-02-08 | Tampering | cobrança/trabalho duplicado após crash | mitigate | requeue_running no startup + UNIQUE(original_hash,step) + dedup gate + content_hash único dos blocos tornam reprocesso no-op idempotente (PROC-03); test_resume_on_startup + test_dedup_gate |
| T-02-09 | Denial of Service | split de PDF grande bloqueia o event loop (API/health congela) | mitigate | split despachado via asyncio.to_thread (Pitfall 4); grep verifica presença |
