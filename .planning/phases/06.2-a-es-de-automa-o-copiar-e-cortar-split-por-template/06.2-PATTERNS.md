## ⚠️ Decisão Arquitetural Crítica (ler antes de planejar)

O modelo da Fase 6 é **single-output por documento**, codificado em 4 lugares:

1. `executor.AutomationPlan` carrega **um** `target_folder` + **um** `target_name`
   (`executor.py:34-50`). `_apply_actions` muta UM nome/UMA pasta em memória e
   devolve UM par (`executor.py:83-118`).
2. `stage.apply_stage` materializa **uma vez** (`materialize_to_dest` único) e escreve
   **um** `AuditLog` write-ahead (`stage.py:530-562`).
3. `AuditLog` é 1 registro = 1 par origem→destino (`audit_log.py:42-44`).
4. `DryRunRow` / `ApplyStageResult` expõem **um** `dest_path` por documento
   (`stage.py:81-102`, `automations.py:364-375`, `types.ts:282-292`).

**Copy + Move coexistindo (D-03) = várias saídas para o mesmo documento.** As cópias
acontecem ANTES do move; cada cópia precisa do SEU write-ahead + verificação de hash
(D-07), e o undo precisa apagar CADA cópia (D-06). Isso NÃO cabe no `AutomationPlan`
de campo único atual.

**Duas abordagens (o planner escolhe):**
- **(A) Plano multi-saída:** `AutomationPlan` passa a carregar uma `list[PlannedOutput]`
  (cada um: kind=`copy|move`, folder, name). `apply_stage` itera: materializa cada cópia
  (sem `remove_original`) → por fim o move (com `remove_original`). N `AuditLog` (um por
  saída, todos no mesmo `run_id`/`document_id`, distinguindo `action="copy"` vs
  `action="apply"`/`move`). `DryRunRow` → `list` de pares por documento. Cobre D-03 de
  fato.
- **(B) Mínimo viável v1:** restringir o v1 a "**ou** copy(s) **ou** move, não ambos no
  mesmo doc encadeado", mantendo o single-output — mas D-03 pede explicitamente
  coexistência ("copiar para Arquivo/ E mover para Processados/"). **Não recomendado**:
  contradiz D-03.

> O `content_hash` no `AuditLog` (`audit_log.py:50`) e o CAS imutável já dão a base
> física para N cópias com undo seguro — a mudança é estrutural (cardinalidade do
> plano/audit), não de mecânica de arquivo. `fileops` já tem tudo.

## Pattern Assignments

### `backend/app/automation/fileops.py` (utility, file-I/O)

**Analog:** `materialize_to_dest` (CAS→destino verificado) — copy reusa ISSO, só NÃO chama `remove_original`.

**O motor seguro JÁ EXISTE e serve copy sem mudança** (`fileops.py:244-259`):
```python
def materialize_to_dest(content_hash: str, dst: Path) -> Path:
    blob = cas.read_bytes(content_hash)
    return _verified_write([blob], Path(dst), content_hash)
```

**`resolve_collision` (anti-colisão D-09/D-10) reusado idêntico** (`fileops.py:91-122`) —
copy NUNCA sobrescreve (D-07). Mesma assinatura `resolve_collision(dst, src)`.

**`remove_original` é o ÚNICO ponto que copy PULA** (`fileops.py:191-198`):
```python
def remove_original(source_path: Path) -> None:
    """Remove o arquivo original ... (AUT-06 crit 5)."""
    Path(source_path).unlink(missing_ok=True)
```
> Copy = `materialize_to_dest(content_hash, dest)` **sem** a chamada subsequente a
> `remove_original`. NENHUMA função nova de fileops é estritamente necessária; a
> diferença mora no orquestrador (`stage.py`). Opcional: uma fachada
> `copy_to_dest = materialize_to_dest` por clareza semântica (mesmo corpo).

### `backend/app/automation/executor.py` (service, transform — PURO, sem disco)

**Analog:** o ramo `elif action.action_type == "move"` em `_apply_actions` (`executor.py:109-114`):
```python
elif action.action_type == "move":
    resolved_folder = naming.resolve_dest_folder(
        action.params.get("dest_folder", ""), fields, base_root=base_root
    )
    if resolved_folder is None:
        return None, None, True  # D-07 / V4
    folder = resolved_folder
# action_type desconhecido: ignora (ação inerte) — falha fechada (V5).
```

**Mudança para `copy`:** `copy` NÃO muta o `(folder, name)` corrente do documento — ele
produz uma **saída adicional** (D-03: cópia vai para SEU `dest_folder` próprio, com o
nome composto pelo rename corrente). Logo `_apply_actions` precisa **acumular saídas**,
não só mutar o par final:
```python
# NOVO contrato sugerido (abordagem A): além de (folder, name, blocked),
# acumular outputs de cópia resolvidos:
elif action.action_type == "copy":
    resolved_folder = naming.resolve_dest_folder(
        action.params.get("dest_folder", ""), fields, base_root=base_root
    )
    if resolved_folder is None:
        return ..., True  # D-07/V4 (mesma regra de bloqueio do move)
    copy_outputs.append(PlannedOutput(kind="copy", folder=resolved_folder, name=name))
```
> O `name` da cópia é o nome-alvo CORRENTE no ponto da ação (rename antes → cópia herda
> o nome renomeado; ordem do D-03). O dispatch desconhecido permanece inerte/falha-fechada
> (`executor.py:116`) — adicionar `copy` é seguro e aditivo (a própria CONTEXT D-01 cita isso).

**`AutomationPlan` (executor.py:34-50)** precisa de um campo `copies: list[...]` (ou a
list[PlannedOutput] da abordagem A). Hoje é `frozen` com 5 campos single-output.

### `backend/app/automation/stage.py` (service orquestrador, event-driven + file-I/O)

**Analog:** o bloco write-ahead → materialize → remove_original do `apply_stage`
(`stage.py:527-562`). É AQUI que a cardinalidade muda. Sequência atual (single):
```python
# (6) WRITE-AHEAD (AUT-04): intenção persistida ANTES de tocar o disco.
audit = AuditLog(
    document_id=doc.id, action=_ACTION, status="intent",
    source_path=str(source), dest_path=str(dest),
    run_id=run_id, content_hash=content_hash,
)
session.add(audit); session.commit(); session.refresh(audit)
# (7) Materializa do CAS e verifica o hash (AUT-06).
fileops.materialize_to_dest(content_hash, dest)
# (8) Verificação passou → remove a origem (AUT-06: copia→verifica→remove).
if physically_moved:
    fileops.remove_original(source)
```

**Padrão de write-ahead a replicar POR CÓPIA (D-07):** para cada saída de cópia, repetir
o passo (6)+(7) — um `AuditLog(status="intent" → "done", action="copy", source_path,
dest_path=<destino da cópia>, content_hash)` ANTES de materializar essa cópia, e **NUNCA**
chamar `remove_original` para cópias. Só o move (se houver) remove a origem, e só **depois**
de todas as cópias verificadas (D-03: "o original é a garantia até todas as saídas estarem
verificadas").

**Anti-colisão por cópia (D-07):** reusar o bloco `resolve_collision` (`stage.py:498-509`)
para cada destino de cópia (idêntico → skip D-10; diferente → sufixo D-09).

**`_plan_dest` (stage.py:305-316)** compõe UM destino do plano; com multi-saída precisa
compor o destino de CADA cópia (mesma lógica de preservar extensão).

**`ApplyStageResult` (stage.py:81-102)** hoje carrega um `dest_path`; multi-saída pede
uma lista de resultados por cópia (ou um agregado). Idempotência (`_has_done`,
`stage.py:319-327`) precisa considerar todas as saídas concluídas, não só uma.

**`dry_run` (stage.py:330-395)** monta UM `ApplyStageResult`; precisa emitir N linhas
(uma por cópia + a do move), cada uma com seu `dest_path` e flags.

**`reconcile_orphans` (stage.py:589-630)** já itera por `AuditLog(status="intent")` —
funciona sem mudança para N intents de cópia (cada um reconciliado pelo seu `dest_path`+hash).

### `backend/app/automation/undo.py` (service, file-I/O)

**Analog:** `undo_operation` move-back + fallback CAS (`undo.py:80-126`):
```python
if dest is not None and dest.exists():
    expected = fileops.hash_file(dest)
    source.parent.mkdir(parents=True, exist_ok=True)
    fileops._verified_write(fileops._stream_file(dest), source, expected)
    dest.unlink(missing_ok=True)
    result = "undone"
else:  # destino sumiu → rede final do CAS
    blob = read_bytes_from_cas(content_hash)
    _atomic_write_bytes(blob, source); result = "undone_from_cas"
```

**Undo de copy (D-06) = APAGAR a cópia, NÃO devolver à origem** (o original nunca foi
tocado). Bem mais simples que o undo de move:
```python
# Ramo novo para AuditLog(action="copy"):
if dest is not None and dest.exists():
    dest.unlink(missing_ok=True)   # apaga a cópia; CAS preserva o conteúdo (D-06)
audit.status = "undone"            # nunca toca a source (original intacto)
```
> Discriminar pelo `audit.action` (`"copy"` vs `"apply"`/move). `undo_document`
> (`undo.py:146-161`) e `undo_run` (`undo.py:164-185`) JÁ iteram todos os `AuditLog(status="done")`
> do doc/run → revertem cópias E move juntos, sem mudança de orquestração (D-06: "undo
> reverte a execução inteira"). O `_reopen_document` (`undo.py:129-144`) também é reusado.

## Shared Patterns

### Write-ahead audit logging (AUT-04) — invocação por saída
**Source:** `backend/app/automation/stage.py:527-541`
**Apply to:** cada saída de cópia (N intents) + o move (D-07).
```python
audit = AuditLog(
    document_id=doc.id, action=_ACTION, status="intent",
    source_path=str(source), dest_path=str(dest),
    run_id=run_id, content_hash=content_hash,
)
session.add(audit); session.commit(); session.refresh(audit)
# ... materialize ... então audit.status = "done"
```
Para copy: `action="copy"` (discrimina no undo); um registro por cópia, todos no mesmo
`run_id`/`document_id`/`content_hash`. O `content_hash` (`audit_log.py:50`) é a rede de
undo via CAS — idêntico para todas as saídas (mesmo conteúdo).

### Anti-colisão (D-09/D-10) — nunca sobrescreve
**Source:** `backend/app/automation/fileops.py:91-122` (`resolve_collision`)
**Apply to:** todo destino de cópia, antes de materializar. Idêntico → skip; diferente → sufixo `_1`.

### Materialização verificada do CAS (AUT-06)
**Source:** `backend/app/automation/fileops.py:244-259` (`materialize_to_dest`) + `:125-182` (`_verified_write`)
**Apply to:** cada cópia (sem `remove_original`) e o move (com `remove_original` só ao final).

### Dispatch falha-fechada (V5)
**Source:** `backend/app/automation/executor.py:116` (comentário "action_type desconhecido: ignora")
+ `backend/app/api/automations.py:59,107-114` (`_ACTION_TYPES` + validador).
**Apply to:** adicionar `copy` ao conjunto E ao executor; sem o ramo seria inerte (seguro).

### Vocabulário visual de badges (design system `--st-*`)
**Source:** `frontend/src/pages/DryRunPage.tsx:17-70` + `AutomationsPage.tsx:71-77` (`ACTION_META`)
**Apply to:** novo estado "copiado / não remove original" reusando tokens existentes
(`--st-encontrado*` p/ cópia segura; cor própria no `ACTION_META.copy`).

Nenhum arquivo está sem analog — `move` é o molde direto de tudo. O único item **sem
