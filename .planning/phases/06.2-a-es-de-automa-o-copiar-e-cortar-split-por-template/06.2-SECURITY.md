## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| cliente/HTTP → `/automations` | body de criação com `action_type` + `dest_folder` controlados pelo usuário | string de configuração (não confiável) |
| `params_json` (dest_folder com tokens) → caminho físico de destino | resolução do destino da cópia contra campos extraídos, materializa no disco | tokens `{campo}` → caminho (não confiável) |
| campos extraídos (IA) → tokens de caminho | valores não confiáveis da IA injetados em `{cliente}` etc. | dados do documento (sensível LGPD) |
| undo → filesystem | apaga arquivos por `dest_path` registrado no AuditLog | caminho de arquivo persistido |
| resposta dry-run/apply → render na UI | paths exibidos na tabela; sem valores de campo | strings de caminho |
