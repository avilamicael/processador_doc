# Phase 06.2 Plan 03: Ação Copiar no frontend (editor + dry-run) Summary

**Ação "Copiar" exposta de ponta a ponta na UI — botão + pasta de destino com tokens e nota "o original permanece" no editor de automações, badge "Copiado — original mantido" no dry-run, tipos TS sincronizados com o discriminador action_kind/removes_original do backend; fluxo criar→dry-run→aplicar→desfazer verificado ao vivo sem perda de arquivos.**

## Performance

- **Duration:** ~25 min
- **Tasks:** 3 (2 auto + 1 checkpoint human-verify)
- **Files modified:** 3

## Accomplishments

- Editor de automações (Condições→Ações) ganhou a ação **Copiar** ao lado de Renomear/Mover, com campo "Pasta de destino" (tokens) e a nota "O original permanece onde está — uma cópia é criada no destino" (D-02/D-05).
- Dry-run renderiza cada saída de cópia como linha origem→destino com o badge **"Copiado — original mantido"** (tooltip "O arquivo será copiado para o destino. O original permanece onde está."), usando tokens --st-* existentes (D-07/AUT-03).
- Tipos TS espelham o backend 1:1: `ActionType` inclui `'copy'`; `DryRunRow` ganhou `action_kind` (`'copy'|'move'|null`) e `removes_original` (`false` na cópia) — `npm run build` verde (gate de drift de tipos D-04).
- copy é estritamente aditivo: rename/move sem regressão na UI.

## Files Created/Modified

- `frontend/src/types.ts` — `ActionType` estendido com `'copy'`; `DryRunRow` ganha o discriminador `action_kind`/`removes_original` (espelha o backend do Plano 02); comentários documentam as N linhas por doc (cópias + move).
- `frontend/src/pages/AutomationsPage.tsx` — `ACTION_META.copy` (label "Copiar", icon folder, dot `--st-encontrado`); barra de botões inclui `'copy'`; `toCreateBody` mapeia copy→`{action_type:'copy', params:{dest_folder}}` preservando a distinção via `a.action_type`; `automationToDraft`/`validate`/`renderAct` generalizados para tratar move|copy igualmente no input "Pasta de destino"; nota D-05 só no caso copy.
- `frontend/src/pages/DryRunPage.tsx` — `SituationBadge` ganha o ramo de cópia (`row.action_kind === 'copy'`) com texto "Copiado — original mantido" e tooltip, antes do default, mantendo a precedência dos ramos blocked/no_match/skipped_identical/collision.

## Decisions Made

- **copy reusa o caminho do move:** mesmo input/validação/mapeamento (`pattern` genérico = `dest_folder`), com `toCreateBody` preservando `a.action_type` — sem ramo paralelo, garantindo zero regressão em rename/move (D-04).
- **Diferenciação visual mínima:** dot de cor `var(--st-encontrado)` para copy (distinto de rename/move) + nota textual "o original permanece" (D-05), sem cores hard-coded fora do design system.
- **Badge dirigido pelo backend:** o dry-run decide o badge por `action_kind`/`removes_original` vindos da API (Plano 02), não por inferência no frontend.
