<objective>
Estender o executor PURO das automações para suportar a ação **Copiar** como
**saída ADICIONAL** por documento — sem tocar o disco, sem ORM, sem migração.

Hoje o modelo é **single-output**: `AutomationPlan` carrega UM `target_folder` +
UM `target_name`, e `_apply_actions` apenas muta esse par em memória. Copiar + Mover
coexistindo (D-03) exige **N saídas por documento**: cada cópia vai para o SEU
`dest_folder`, com o nome-alvo corrente, enquanto o move (se houver) continua sendo
o plano-alvo final. Esta é a **decisão arquitetural crítica** da fase (ver
06.2-PATTERNS.md §"Decisão Arquitetural Crítica", abordagem A).

Purpose: criar o contrato puro (multi-saída) que o `stage` (Plano 02) vai
materializar — definir interfaces ANTES de quem as consome (interface-first).
Output: `AutomationPlan` com lista de cópias planejadas; `_apply_actions` que acumula
cópias; docstring de modelo atualizado; testes RED→GREEN fixando o contrato.
</objective>
