# Phase 06.1 Plan 02: API de Templates com Sinais em Grupos E/OU Summary

**API `/templates` adaptada ao novo schema de sinais como GRUPOS E/OU (D-T2): schema Pydantic `In/Patch/Out` com `signals: list[list[SignalConditionIn]]`, serialização forward-compatible para `signals_json` (lê forma plana legada), `mode` validado por `Literal` (422 sem eval), regex string-only (T-04-10) e `doc_type` rebaixado a coluna dormente (D-T5) — sem migração de schema.**

## Accomplishments
- `SignalConditionIn` (`mode: Literal["texto","regex"]`, `value: str` com `field_validator` que rejeita vazio/só-espaços) e `SignalConditionOut` espelho de resposta. O `Literal` faz o dispatch por etiqueta sem `eval` (V5) e devolve 422 limpo em mode desconhecido (T-06.1-07).
- `signals: list[list[SignalConditionIn]]` em `TemplateIn`/`TemplatePatch`; `TemplateOut.signals: list[list[SignalConditionOut]]`.
- `_loads_signals` (plano, legado) substituído por `_loads_signals_groups` — réplica EXATA de `matcher._parse_groups`: JSON inválido → `[]`; forma plana `list[str]` → 1 grupo OU por termo; grupos normalizados mantendo só `dict`. Endpoint e matcher leem a MESMA forma canônica.
- `_dump_signals` serializa os modelos Pydantic → `[[{mode,value},...],...]` (mesma forma que o matcher consome) em `create_template`/`update_template`.
- `doc_type` mantido no schema como legado/opcional (D-T5); docstrings de `api/templates.py` e `models/template.py` atualizadas (grupos E/OU + doc_type dormente). NENHUMA mudança de `mapped_column`, NENHUMA migração nova.
- Regex string-only confirmada (T-04-10): sem `import re`, sem `re.compile`/`re.search`/`re.fullmatch` executáveis (as 3 ocorrências de grep são prosa da docstring de segurança).
- Suite `test_api_templates.py` estendida (CRUD/patch atualizados + 7 novos testes); 30 testes do arquivo verdes, suite backend completa 373 verdes, `test_migrations.py` verde (trigger `trg_documents_updated_at` intacto).

## Decisions Made
- **Forma canônica única lida/escrita pelos dois lados:** `_loads_signals_groups` no endpoint replica `matcher._parse_groups`; o seam de serialização fica sincronizado (mudança em um exige mudança no outro — documentado nas docstrings).
- **`doc_type` dormente, não removido (D-T5):** mantido no schema In/Patch por compat e no modelo como coluna; nenhuma feature o alimenta, nenhuma migração de remoção — evita tocar `documents` e o trigger.
- **Regex string-only no endpoint (T-04-10/T-06.1-06):** endpoint não importa `re`; regex catastrófica persiste como string (testado com `(a+)+$` → 201). Execução segura só no matcher (Plano 01) com tetos ReDoS.
- **`mode` por `Literal` (T-06.1-07):** dispatch por etiqueta no Pydantic; mode inválido → 422 sem `eval` (V5), nunca 500.
