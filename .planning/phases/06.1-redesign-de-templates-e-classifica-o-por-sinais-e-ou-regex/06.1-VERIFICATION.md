## Gaps Summary

### Bloqueio Principal: CR-01 — Defesa ReDoS insuficiente

**Root cause:** O teto de input (`_MAX_HAYSTACK_LEN = 200_000`) apenas limita o número de
caracteres passados ao `re.search`. O custo de backtracking exponencial é determinado pela
**estrutura local do pattern** — um pattern de 5 caracteres como `(a+)+$` contra 41 caracteres
de input trava o engine indefinidamente, independentemente do tamanho total do haystack.

**Evidência da verificação independente:**
- Pattern: `(a+)+$` → 5 chars, bem abaixo de `_MAX_SIGNAL_REGEX_LEN=512`
- Haystack: `'a'*40+'X'` → 41 chars, muito abaixo de `_MAX_HAYSTACK_LEN=200_000`
- Resultado: processo morto por timeout de 3 segundos (exit 124)
- Confirmação do caso bom (não patológico): `'a'*200_000` (all-a matching) completa em 0.0004s — este é o caso que `test_redos_nao_trava` usa, dando falsa cobertura

**Vetores reais:** O operador cola o pattern `(a+)+$` no form de sinais (aceito como string
por design). O `full_text` de um documento contendo uma sequência de letras repetidas seguida
de qualquer caractere diferente dispara o cenário patológico, travando o worker de
classificação.

**Fix prescrito (do REVIEW CR-01):**
```python
import regex  # pip install regex

_REGEX_TIMEOUT_S = 0.25

def _condition_matches(cond: dict, haystack: str) -> bool:
    value = str(cond.get("value", ""))
    mode = cond.get("mode", "texto")
    if mode == "regex":
        if not value or len(value) > _MAX_SIGNAL_REGEX_LEN:
            return False
        try:
            pattern = regex.compile(value, regex.IGNORECASE)
            return pattern.search(
                haystack[:_MAX_HAYSTACK_LEN], timeout=_REGEX_TIMEOUT_S
            ) is not None
        except (regex.error, TimeoutError):
            return False  # falha fechada
    # ...
```

**Items do gap:**
1. Substituir `re` por `regex` com timeout no `_condition_matches`, OU implementar execução
   com `concurrent.futures.ThreadPoolExecutor` com deadline
2. Fortalecer `test_redos_nao_trava` para usar o caso patológico (`'a'*40+'X'`) com
   asserção de tempo (`time.monotonic() < deadline`)
3. Corrigir docstring (linhas 22-25, 48-51) para remover a afirmação falsa de que o teto
   de input neutraliza ReDoS
