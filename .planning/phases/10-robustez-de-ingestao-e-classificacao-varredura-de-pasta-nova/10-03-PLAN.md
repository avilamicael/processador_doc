<objective>
Adicionar a ação "reprocessar automático" (D-10..D-12): re-roda matcher→(IA)→filler com os
templates ATUAIS, SEM forçar template, para tirar docs da QUARENTENA/EM_REVISAO após o
usuário editar um template — sem deletar+re-ingerir o arquivo. Por-documento E em lote
(por balde). Espelha o `reclassify_document` existente, mas SEM `forced_template_id`.

Purpose: Resolver o fluxo "ajustei o template e o doc continua na quarentena". `classify_stage`
já recarrega templates do DB a cada run, então o reprocess pega as edições de graça.
Output: 2 endpoints novos em `documents.py` + cobertura em `test_api_documents.py`.

DECISÃO DE PLANNING: batch por **`bucket`** ("quarentena"|"em_revisao") — o backend resolve
os ids a partir do mesmo filtro de `get_attention` (botão "reprocessar todos", D-12).
Também aceitar `ids` explícitos como alternativa, ignorando ids fora dos estados elegíveis.
</objective>

| cliente → POST /documents/{id}/reprocess + /documents/reprocess | request muda estado de documento e enfileira trabalho |
| reprocess → state machine / fila | transição de estado + requeue idempotente |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-05 | Elevation of Privilege | guard de estado do reprocess | mitigate | Guard semântico explícito `state not in {QUARENTENA, EM_REVISAO}` → 409 ANTES do transition (Pitfall 4); a allowlist da state machine sozinha não basta (PROCESSANDO é alcançável de vários estados) — não permitir escalonar estado fora da allowlist |
| T-10-05D | Denial of Service | batch reprocess | mitigate | Batch numa só sessão; ids fora dos estados ignorados (idempotente); requeue UNIQUE-safe via `_requeue` (não duplica Job). Re-enfileirar `classify` não dispara IA por si só (matcher local custo-zero primeiro) |
| T-10-05I | Information Disclosure | logs do reprocess | mitigate | Logar só metadados (document_id/estado); NUNCA `full_text`/valores (V7) |
| T-10-05T | Tampering | idempotência do stage | mitigate | Apagar `ClassificationResult` antes do requeue (Pitfall 3) garante que o reprocess realmente reclassifica e não deixa estado inconsistente (doc em PROCESSANDO sem CR a reprocessar) |
| T-10-SC | Tampering | npm/pip installs | accept | Nenhum pacote novo neste plano. Sem checkpoint de instalação |
