<objective>
Expor na UI as três capacidades backend: (1) painel "Testar sinais" no construtor de templates
(upload de PDF → relatório por-grupo/sinal, com aviso de escaneado — D-07/D-08/D-09);
(2) botões "Reprocessar" (por-doc) e "Reprocessar todos" (por balde) na visão de atenção

Purpose: Fechar a entrega da fase — o usuário vê quais sinais falham, recupera docs da
quarentena após editar templates, e liga (opcionalmente) a IA-fallback de forma consciente.
Output: types/api/hooks + 3 páginas atualizadas + verificação visual ao vivo.

DECISÃO DE PLANNING: preview via **base64** no body JSON (sem multipart) — a função
`previewSignals` lê o `File` via `FileReader`/`arrayBuffer→base64` e envia `{template_id, pdf_base64}`
pelo `request<T>` JSON-only existente. Zero npm novo.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| navegador → backend (preview/reprocess/config) | a UI envia PDF base64, ids/buckets e o toggle; o backend (Planos 02/03/04) valida tudo |
| backend → DOM | nomes/valores de sinal e do documento são renderizados na tela |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-XSS | Information Disclosure / Tampering | render de valores no DOM | mitigate | Renderizar nomes/valores como texto puro (React escapa por padrão); NUNCA `dangerouslySetInnerHTML` (T-02-11) |
| T-10-04F | Denial of Service | upload do preview no cliente | mitigate | Validação autoritativa no backend (teto de bytes/magic bytes, Plano 02); a UI apenas envia base64. Opcional: rejeitar arquivos obviamente grandes antes do envio |
| T-10-03U | Custo | toggle IA-fallback na UI | mitigate | Aviso explícito de custo por chamada de IA ao lado do toggle; default OFF refletido do backend (Plano 04) |
| T-10-05U | Elevation of Privilege | reprocessar na UI | accept | A UI só dispara o reprocess; os guards 409 (estado elegível) são autoritativos no backend (Plano 03). UI não decide estado |
| T-10-SC | Tampering | npm installs | accept | Nenhum pacote npm novo (FileReader/btoa nativos do browser; TanStack Query já presente). Sem checkpoint de instalação |
</threat_model>
