<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API /templates | Body de template (nomes, regex, sinais) é input não confiável |
| client → API /documents/{id} | document_id na rota é input não confiável |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Tampering / Injection | SQL injection em /templates e /documents/{id} | mitigate | SQLAlchemy ORM/Core parametrizado (sem string-building); `document_id` tipado int na rota; padrão já em uso |
| T-04-10 | DoS | regex do operador armazenada via /templates | mitigate | O endpoint apenas ARMAZENA a regex como string; NÃO compila/executa (a execução segura — fullmatch + limite de tamanho — é o Plan 02). Sem ReDoS no caminho HTTP |
| T-04-11 | Information Disclosure | detalhe do doc vaza dados sensíveis em log | mitigate | Endpoint loga só metadados (document_id/template_id); valores extraídos vão no corpo da resposta (consumo legítimo da UI), nunca em log (V7/V8) |
| T-04-12 | Tampering | XSS via nomes de template/valores no frontend | mitigate | React renderiza como texto puro (T-02-11 estabelecido); o backend não emite HTML — reforçado no Plan 06 |
</threat_model>
