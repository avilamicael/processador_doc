## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| valor extraído (documento) → validador | Conteúdo do documento é input não confiável |
| regex do operador → engine de regex | Pattern fornecido pelo operador (single-tenant) roda contra o valor |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03 | Denial of Service | regex do operador (D-09) em fields.validate_field | mitigate | `re.fullmatch` sobre valor com teto de tamanho (ex. 4096 chars) antes de aplicar; pattern compilado por chamada, não em loop quente; operador é single-tenant (não atacante externo) → risco baixo, mas limitado por design (V5 / 04-RESEARCH.md Security Domain) |
| T-04-04 | Information Disclosure | logs do módulo de validação | accept | Módulo puro sem logging de valores; se logar, só metadados (field_type/valid), nunca o valor — segue disciplina V7/V8 da Fase 3 |
| T-04-05 | Tampering | parse incorreto silencioso (float em moeda, dia↔mês) | mitigate | Decimal obrigatório para moeda/número; `dayfirst=True` obrigatório para data; parse falho → None + marca inválida (nunca chuta um valor errado) |
</threat_model>
