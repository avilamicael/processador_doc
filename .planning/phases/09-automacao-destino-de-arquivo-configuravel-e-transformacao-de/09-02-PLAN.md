<objective>
Implementar o ENGINE DE TRANSFORMAÇÃO de valores (backlog item 11): filtros inline
encadeáveis no token (`{campo:filtro=arg:filtro}`, D-06) com o conjunto v1 (D-07):
`palavras=N`, `letras=N`/`truncar=N`, `maiusc`, `minusc`, `sem_acento`, `padrao=…`,
`substituir=de>para`, e `formato=` (expondo o `_fmt_date` existente). A sanitização de
chars proibidos do Windows continua automática, aplicada DEPOIS das transformações
(D-08, por segmento).

Purpose: hoje o nome/pasta só aceita `{campo}` cru + sanitize — não dá para encurtar
"IGUACU DIST. DE PROD. OTICOS LTDA" para "IGUACU". Os filtros tornam o nome
transformável como o usuário deseja.
Output: `_substitute` com um pipeline de filtros (dispatch explícito, nunca `eval`),
afetando tanto `name_pattern` (rename) quanto os segmentos de `dest_folder` (move/copy).
</objective>

  <implementation>
    Estenda `_substitute` (dentro de `repl`), após obter o `value` cru do campo:
    1. Se `spec is None` → nenhum filtro (caminho atual intacto, retrocompat de `{campo}`).
    2. Se o spec NÃO contém `=` E casa o atalho de data (contém `aaaa`/`mm`/`dd`) → `_fmt_date(value, spec)` (legado; None → _MissingField como hoje).
    3. Caso contrário → PIPELINE: dividir `spec` por `:` numa lista de filtros e aplicar em ordem, com DISPATCH EXPLÍCITO por nome (NUNCA `eval`/`format` dinâmico). Para cada filtro `f`:
       - `palavras=N` → `" ".join(value.split()[:int(N)])`
       - `letras=N` / `truncar=N` → `value[:int(N)]`
       - `maiusc` → `value.upper()` ; `minusc` → `value.lower()`
       - `sem_acento` → `_strip_accents(value)`
       - `substituir=de>para` → `de, _, para = arg.partition(">"); value.replace(de, para)`
       - `formato=spec` → `_fmt_date(value, spec)` (None → _MissingField: pediu formato e o valor não é data)
       - `padrao=arg` → tratado ESPECIAL (ver abaixo)
       - filtro desconhecido / `int()` inválido → INERTE (não altera o value; nunca eval) — falha-fechada amigável.
    `padrao=` (D-07 / RESEARCH Pattern 4 / A3): o bloqueio por campo ausente acontece HOJE antes de chegar ao spec. Confirme com o teste `test_filter_padrao_default_when_missing`.
    Adicione `import unicodedata` no topo e o helper `_strip_accents(s: str) -> str` (RESEARCH Code Examples).
    `_TOKEN_RE`: o group(2) atual já captura tudo após o 1º `:` (inclusive `:` internos), então o split por `:` dentro do `repl` é suficiente — NÃO precisa mudar a regex (confirme com `test_filter_chain`). NÃO logar valores (V7/V9). A sanitização permanece a cargo de `resolve_pattern`/`resolve_dest_folder` (sanitize_component DEPOIS, D-08) — NÃO sanitizar dentro do pipeline.
