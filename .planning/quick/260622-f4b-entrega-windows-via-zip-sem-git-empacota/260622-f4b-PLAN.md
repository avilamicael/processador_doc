<objective>
Habilitar a entrega e atualização do sistema no Windows via **ZIP auto-contido** publicado em **GitHub Releases**, sem que o cliente piloto precise de Git nem de Node.

Purpose: O cliente piloto recebe um pacote `.zip` com o `frontend/dist` JÁ buildado; instala com `instalar.ps1` (que pula o build) e atualiza com `atualizar.ps1` (que baixa o ZIP da última release ou usa um ZIP local). O artefato de build continua git-ignored — só existe dentro do ZIP de release.

Output:
- `empacotar.ps1` (NOVO) — roda pelo DEV: builda o frontend e monta o ZIP de release versionado.
- `atualizar.ps1` (REESCRITO) — atualizador sem Git, via GitHub Releases ou `-LocalZip`.
- `instalar.ps1` (AJUSTE) — pula o build quando `frontend\dist` já existe.
- `INSTALL-WINDOWS.md` (ATUALIZADO) — dois fluxos documentados (cliente / dev).
</objective>
