# Spec de Copy — refatoração de linguagem do frontend (contrato único)

Data: 2026-06-26. Base: auditoria em `2026-06-26-copy-audit-frontend.md`.
Decisões do usuário: (1) terminologia **plena** (trocar todo o jargão); (2) exemplos de **domínios variados** neutros; (3) TemplatesPage = **copy + suavizar estrutura**.

**Objetivo:** linguagem clara e didática para um usuário LEIGO (não-técnico), e GENÉRICA (qualquer tipo de documento — nota fiscal é só um exemplo entre vários). Toda string voltada ao usuário deve seguir este contrato. Responder/escrever em **pt-BR**.

---

## 1. Regras de tom (aplicar em tudo)

- **Sem jargão técnico.** Nada de "template", "sinais", "matcher", "regex" (exceto rótulo do modo avançado), "quarentena", "ingestão", "dry-run", "watcher", "limiar", "fallback", "pipeline", "token", "normalizado", "lote/run", "OCR/deskew" cru.
- **Voz ativa, 2ª pessoa.** Fale com o usuário ("você"), direto e curto. Frases de até ~2 linhas.
- **Tranquilizar onde há risco.** Manter o tom de "nada se perde / o original nunca é tocado" já presente (NÃO regredir esse tom).
- **Não inventar funcionalidade.** Só reescrever o texto; o comportamento é o mesmo.
- **Preservar o que já está bom** (NÃO mexer no tom): copy de reversibilidade da DryRunPage, confirmações destrutivas ("nenhum arquivo se perde"), estados vazios de Documentos, badges de validação ("válido/inválido").

## 3. Exemplos neutros (domínios variados) — usar este conjunto

Sempre que um exemplo for necessário, usar tipos comuns e variados (NÃO nota fiscal fixa, NUNCA emissores reais tipo "TryLab"/"IGUACU DIST"):

- **Tipos de documento de exemplo:** Recibo, Contrato, Boleto, Holerite, Currículo, Relatório, Comprovante.
- **Campos de exemplo genéricos:** Nome, Data, Valor, Número do documento, Cliente, Empresa.
- **Exemplo de "nome do arquivo":** `{Cliente}_{Data}` → `Maria Souza_2026-01-15.pdf` (neutro).
- **Exemplo de pasta destino:** `C:\Documentos\{Cliente}\` ou `Recibos/{Empresa}/`.
- **Exemplo de "como reconhecer" (texto):** uma palavra/frase que aparece no documento, ex.: `RECIBO`, `CONTRATO DE PRESTAÇÃO`.
- **Exemplo de padrão avançado (regex), só no modo avançado:** `\d{2}/\d{4}` (mês/ano) — neutro, não a chave de 44 dígitos.
- **Exemplo de ajuste do texto:** `{Cliente:maiúsculas:2 primeiras palavras}` → genérico.
- **Dica para a IA (exemplo):** "o valor que aparece após 'Total'", "o nome no topo da página".

Regra: rotacionar os domínios (não repetir sempre "Recibo"); variar para reforçar que serve a qualquer documento.

## 4. TemplatesPage — suavização de estrutura (além do copy)

Manter os dados/comportamento; mudar só apresentação:
- **Esconder "regex" atrás de "Avançado".** O seletor de condição mostra por padrão só **"contém o texto"**. A opção de padrão avançado (regex) fica sob um link/disclosure "Avançado" (ou um toggle "usar padrão avançado"). O termo "regex" só aparece dentro desse modo avançado, rotulado "padrão avançado".
- **Trocar a lógica E/OU exposta** por linguagem simples:
  - "+ E — adicionar condição" → **"E também… (adicionar exigência)"** ou botão "Adicionar outra pista".
  - "+ OU — adicionar grupo" → **"OU de outro jeito… (adicionar alternativa)"**.
  - "Grupo N — todas (E)" → **"Reconhecer se TODAS estas pistas aparecerem"**; entre grupos, deixar claro "OU".
  - A frase "se bater em qualquer grupo (OU); dentro do grupo, todas (E)" → reescrever como: **"O documento é deste tipo se aparecer QUALQUER um dos conjuntos de pistas abaixo. Dentro de um conjunto, TODAS as pistas precisam aparecer."**
- **"Passo 1 / Passo 2"**: manter a ideia de 2 passos, mas explicar sem "sem IA/com IA" cru: Passo 1 = "Como reconhecer este tipo (de graça, sem IA)"; Passo 2 = "Quais dados ler (a IA lê e preenche)".
- **"Testar sinais"** → **"Testar com um arquivo de exemplo"**: "Envie um PDF que já tenha texto para ver se este tipo seria reconhecido — sem custo. Útil para entender por que um documento não foi reconhecido."

## 5. Padrões de mensagem reutilizáveis

- **Erro de carregamento (todas as telas):** "Não foi possível carregar {X}. Verifique se o aplicativo está aberto e tente de novo."
- **Vazio de identificação:** "Nenhum tipo de documento reconheceu este arquivo. Ele fica separado para você conferir — e nunca é apagado."
- **Confirmação de "tentar identificar de novo" (com correções):** "Isto vai ler e identificar o documento de novo e **descartar as correções que você fez à mão**. Continuar?"

## 6. Fora de escopo (não mexer)
- Nomes internos (localStorage `docwatch-theme`), nome do produto "DocWatch", "Gestão documental".
- Telas marcadas "em breve" (Regras de separação, Integrações, opções de OCR mock): suavizar só o que estiver muito técnico, sem investir.
- Lógica, hooks, endpoints, tipos — só texto e (no Templates) a apresentação das condições.
